# AGENTS.md

本文件给以后处理本仓库任务的 Codex / Claude 使用。请先读完本文件，再开始改代码或文档。

## 项目定位

这是一个大一 C 语言课程项目：命令行图书管理系统。

项目代码应保持简单、直接，适合初学者阅读和理解。不要把项目改成复杂的企业级结构，也不要引入和课程项目规模不匹配的设计。

## 基本原则

- 每次只处理当前 Issue。
- 不要顺手做额外功能。
- 不要大规模重构，除非 Issue 明确要求。
- 不要引入第三方依赖。
- 不要改变 `books.txt` 当前保存格式，除非 Issue 明确要求。

## Git 工作流

- 每个 Issue 使用独立分支。
- 每个 PR 只解决一个明确问题。
- 不要直接提交 `main`。
- PR 描述要说明修改内容和测试方式。
- 如果 PR 解决某个 Issue，要在描述中写 `Closes #编号`。

## 代码修改要求

- 修改前先理解现有函数和数据结构。
- 优先做小改动，不要重写整个文件。
- 不能破坏已有功能：添加、显示、查询、删除、修改、借书、还书、预算推荐。
- 修改输入、文件读写、库存逻辑时要特别小心。

## 测试要求

代码修改后必须运行：

```sh
make
./tests/run_all.sh
```

涉及 `books.txt` 的测试必须做到：

- 测试前备份原文件。
- 测试后恢复原文件。
- 不留下 `.bak` 临时文件。
- 不破坏用户真实数据。

## 文档要求

- 文档语言自然，不要太像 AI 模板。
- 内容符合大一 C 语言项目实际水平。
- 不夸大项目复杂度。

## 完成任务后的输出

任务完成后，请说明：

- 修改了哪些文件。
- 做了哪些改动。
- 运行了哪些测试。
- 测试结果是否通过。
- 是否需要创建 PR。
